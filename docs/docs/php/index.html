PHP Coding Conventions

Naming

  1. Self-explanatory names
  2. If context is given, use short names
  3. Use positive logic (e.g. is_visible instead of is_hidden)
  4. Don't use abbreviations
  5. Spell check
  6. Always start a Boolean with 'is' or 'has'

Comments

We use comments:

  • to explain why a block of code does not seem to fit conventions
  • describe workarounds with//WORKAROUND:

Strings

When possible prefer string interpolation above sprintf and the . operator.

✔

$greeting = "Hi, I am {$name}.";

✖

$greeting = 'Hi, I am ' . $name . '.';

String must be surrounded by single quotes

✔

$text = 'Today is Monday'

✖

$text = "Today is Monday"

Functions

String must be surrounded by single quotes

✔

public function firstFunction(Voucher $voucher): void
{
}

public function secondFunction(Voucher $voucher): void
{
}

✖

public function firstFunction(Voucher $voucher): void
{
}
public function secondFunction(Voucher $voucher): void
{
}
